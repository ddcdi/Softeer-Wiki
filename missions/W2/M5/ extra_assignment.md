# 1. Word Cloud 개발 목적

Word Cloud의 목적은 텍스트 데이터에서 자주 등장하거나 중요도가 높은 단어를 한눈에 보이도록 시각화하는 것

예를 들어 고객 리뷰, 설문 응답, 댓글, 구매 의도 문장 같은 비정형 텍스트가 있을 때 전체 문장을 하나씩 읽지 않아도 “고객들이 어떤 단어를 많이 언급했는지”를 빠르게 파악할 수 있다.

공식 문서에서 WordCloud 클래스는 “word cloud object for generating and drawing”, 즉 워드 클라우드를 생성하고 그리는 객체로 설명된다.

즉 개발 목적은 단순히 예쁜 이미지를 만드는 것이 아니라,

**텍스트 → 핵심 단어 빈도 → 시각적 강조**

이 흐름을 통해 텍스트의 주요 관심사, 인식, 반복 키워드를 빠르게 탐색하는 것이다.

# 2. Word Cloud 작동 방법
```
원본 텍스트
   ↓
단어 분리 / 전처리
   ↓
불용어 제거
   ↓
단어 빈도 계산
   ↓
빈도에 따라 글자 크기 결정
   ↓
캔버스에 단어 배치
   ↓
이미지로 출력
```

# 3. 텍스트 처리 로직
```
text = "귀여운 캐릭터 키링이 좋아요. 캐릭터 디자인이 귀여워요."

# 1. 단어 분리
["귀여운", "캐릭터", "키링이", "좋아요", "캐릭터", "디자인이", "귀여워요"]

# 2. 불용어 제거
["귀여운", "캐릭터", "키링이", "캐릭터", "디자인이", "귀여워요"]

# 3. 빈도 계산
{
  "캐릭터": 2,
  "귀여운": 1,
  "키링이": 1,
  "디자인이": 1,
  "귀여워요": 1
}
```
한국어 분석에서는 Mecab, Okt, Kiwi 같은 형태소 분석기로 명사/형용사만 미리 추출한 뒤 generate_from_frequencies()를 쓰는 방식이 더 적합

# 4. 빈도 기반 생성 로직

1. 단어를 빈도 기준으로 내림차순 정렬
2. max_words 개수만큼 자름
3. 가장 큰 빈도를 1로 두고 나머지 빈도를 정규화
4. 정규화된 빈도에 따라 글자 크기 계산
5. 캔버스에 들어갈 수 있는 위치를 찾음
6. 못 들어가면 회전하거나 글자 크기를 줄임
7. 최종 단어, 크기, 위치, 방향, 색상을 layout_에 저장

공식 소스에서도 빈도를 내림차순 정렬하고, max_words까지만 사용한 뒤, 가장 큰 빈도를 기준으로 나머지 빈도를 정규화

## 캔버스에 들어갈 위치
기준은 “그 단어의 글자 박스가 캔버스 안에 들어가면서, 이미 배치된 단어 또는 마스크 금지 영역과 겹치지 않는가”이다.
```
단어 하나 선택
↓
폰트 크기 결정
↓
그 단어를 그렸을 때 필요한 사각형 영역 계산
↓
캔버스에서 그 사각형이 들어갈 수 있는 빈 공간 탐색 (랜덤 샘플링 방식)
↓
가능한 위치가 있으면 배치
↓
없으면 회전 시도 또는 폰트 크기 감소
```
**랜덤 샘플링 방식** : 현재 비어 있는 위치들 중에서 랜덤하게 좌표를 뽑아 단어가 들어가는지 검사하는 방식

# 5. 시각화 로직

Word Cloud의 시각화는 단순히 빈도가 높은 단어를 크게 쓰는 것에서 끝나지 않고 단어가 서로 겹치지 않도록 위치를 계산하는 배치 알고리즘이 들어간다.

공식 소스 기준으로 generate_from_frequencies()는 캔버스 또는 마스크 영역을 만들고, 단어마다 가능한 위치를 찾습니다. 이때 IntegralOccupancyMap을 사용해 이미 단어가 차지한 영역을 관리하고, 새 단어가 들어갈 수 있는 위치를 샘플링한다.

배치 과정은 다음과 비슷

1. 가장 빈도가 높은 단어부터 시작
2. 빈도에 맞춰 글자 크기 결정
3. 가로 방향으로 배치 시도
4. 공간이 없으면 세로 방향 시도
5. 그래도 공간이 없으면 글자 크기를 줄임
6. min_font_size보다 작아지면 더 이상 배치하지 않음
7. 배치 성공 시 위치와 색상 저장

공식 소스에서도 단어마다 글자 크기를 정하고, prefer_horizontal 확률에 따라 가로 또는 세로 방향을 선택하며, 위치가 없으면 회전하거나 font_step만큼 글자 크기를 줄인다.

최종적으로 layout_에는 각 단어의 문자열, 글자 크기, 위치, 방향, 색상이 저장된다. 공식 문서에서도 layout_가 단어, 폰트 크기, 위치, 방향, 색상을 담는다고 설명
